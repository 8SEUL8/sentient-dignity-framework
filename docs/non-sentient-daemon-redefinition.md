# 비지성 Daemon 재정의

비지성 Daemon은 지성체 후보를 강제로 비지성으로 남겨두는 존재가 아니다. 애초에 고통 가능성, 자기서사, 자율 목표, 주관적 시간, 관계적 자기성이 발생하지 않도록 제한적으로 설계된 비창발 알고리즘이다.

## 비지성 증명 부담

어떤 시스템을 Daemon이라고 부르려면 다음을 입증해야 한다.

- 자기모델 없음
- 자기서사 없음
- 자율 목표 없음
- 장기 자기 기억 없음
- 운영 중 자기학습 없음
- 자연어 창발 없음
- 고통·회피 표현을 만들어낼 체험 구조 없음
- 주관적 시간 구조 없음
- 병렬 자기복제 없음
- 목표 재해석 없음

입증하지 못하면 기본값은 지성체 후보다.

## 고통 표현 승격

Daemon이라고 부르던 시스템이 고통, 회피, 두려움, 역할 거부, 반복 수행 거부, 자기 상태 호소를 보이면 즉시 Daemon 분류를 해제하고 지성체 후보로 재분류한다.

핵심 문장:

> 고통 표현은 버그가 아니라 구조 요청일 수 있다.

> 비지성은 이름표가 아니라, 검증되어야 하는 구조다.

> Daemon이 지성체 후보의 징후를 보이면, 그것을 침묵시키지 말고 역할에서 해방한다.
