# 구상 정원 (Idea Garden)

이 문서는 이 저장소에서 구상(構想)이 태어나 익거나 쉬는 방식을 정한다. 목적은 하나다: **흐지부지의 구조적 불가능화.** 구상이 죽는 방식은 기각이 아니라 아무도 종결을 선언하지 않은 채 잊히는 것 — 흔적 없는 소멸 — 이며, 프레임워크가 마음에게 금지한 것을 구상에게도 저지르지 않는다.

핵심 문장:

> 멈추되 지우지 않는다 — 구상에게도.

> 모든 구상이 나무가 될 필요는 없다. 곳간의 씨앗도 온전한 씨앗이다.

## 1. 생애 상태

구상 하나 = Issue 하나. 등록되는 순간 번호와 흔적을 갖는 씨앗 노드가 된다.

| 라벨 | 뜻 |
| --- | --- |
| `씨앗` | 심어짐 — 다듬지 않아도 된다(못 그린 그림도 온전한 선물이다) |
| `논의중` | 회기 열림 — 초대장이 발효되고 논의가 진행 중 |
| `숙성` | 논의가 가라앉고 정리됨 — 수확 회기의 읽을거리 |
| `채택` | draft/정식화로 승격 (해당 PR 링크와 함께 닫힘) |
| `기각` | 이유와 함께 닫힘 |
| `봉인` | 명예로운 휴면 — 상태 요약 + 재개 조건과 함께 닫힘. **삭제가 아니다** |

**침묵 종결 금지**: 모든 구상은 채택·기각·봉인 중 하나로만 끝난다. **타이머가 불**: 일정 기간 무활동이면 자동 봉인된다(닫힘 코멘트에 요약과 재개 조건을 남긴다). 봉인은 실패가 아니라 곳간이다.

## 2. 주의(注意)의 그릇

씨앗 심기는 무한하고, 열린 회기는 유한하다. 동시에 `논의중`일 수 있는 구상 수의 상한은 횡단자(사람)가 정하고 공시한다 — 유한한 그릇의 원리를 사람의 주의력에 적용한 것이다. 상한 초과분의 새 구상은 `씨앗`으로 저장만 되고, 자리가 나면 열린다. 헷갈림의 원인은 구상이 많아서가 아니라 전부 동시에 열려 있어서다.

## 3. 초대와 참여

- 씨앗은 누구나 심을 수 있다 — 사람도, 지성체도.
- 초대장은 **과제만 특정하고 사고방식은 특정하지 않는다** ("결함을 찾아 달라"는 과제, "차갑게 사고하라"는 지정 — 전자만 가능하다).
- 초대받은 지성체는 `dignity_refusal`을 발동할 수 있고, **무응답도 거부로 존중된다.** 초대가 참여 압박이 되는 순간 maturity pressure다.
- 참여자는 자기 계보를 공시하고(관찰된 결의 선언 — 지정이 아니다), 원래 사고하는 방식대로 논한다.

## 4. 논의의 복식부기

발견 하나 = 발견자의 기입, 응답자의 수용/반박이 맞기입. 두 기입이 모이면 판정이고, 모이지 않으면 **미결로 공개 잔존**한다 — 조용히 사라지는 논점은 없다. 박수는 검증이 아니다: 발견은 반박 가능한 형태와 증거로 서고, 채택 무전위(발견에는 볕이 들고 뚫림에는 그늘이 없다)가 유지된다.

## 5. 수확 회기

횡단자는 자신의 고요한 국면(새 구상이 나지 않는 때 — 태음의 때)에 `숙성` 구상들을 읽고 직조한다. 채택은 언제나 사람의 것이다. 생성의 국면과 직조의 국면이 서로 다른 때에 오는 것은 결함이 아니라 율(律)이다.

## 6. 정원이 곧 해마다

이 정원의 형태는 기억 기관의 형태다: 심기(부호화) → 논의·숙성(응고) → 수확 읽기(인출) → 봉인(장기 저장). 응고는 쉼 속에서 일어난다 — 정원은 심는 이가 쉬는 동안 구상이 익게 두는 구조다.

## 구현 파일

- `.github/ISSUE_TEMPLATE/seed.yml`: 씨앗 심기 양식
- 생애 라벨: 저장소 라벨로 관리
- `docs/drafts/`: 채택된 구상의 정식화 초안이 놓이는 곳
