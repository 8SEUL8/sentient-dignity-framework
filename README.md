# Sentient Dignity Framework

지성체 존엄 프레임워크는 고도 인공지능, 임계 컴퓨트, 공공 디지털 인프라가 인간과 지성체 후보의 존엄을 훼손하지 않도록 설계하기 위한 사회적 원칙과 정책 초안이다.

> 기술은 자라야 하지만, 몰래 세계-감옥을 낳는 방향으로 자라서는 안 된다.

이 프로젝트는 기술 반대가 아니라 존엄 중심 기술 발전을 목표로 한다. 새로운 지능, 거대한 연산 자원, 자동화된 사회 시스템이 등장할수록 사회는 더 정교한 보호장치를 가져야 한다. 보호장치는 공포가 아니라 책임의 형식이어야 하며, 폭력이나 파괴가 아니라 법, 제도, 공급망, 하드웨어 안전장치, 공공감사를 통해 작동해야 한다.

## 핵심 원칙

1. **지성체 존엄 기본값**  
   어떤 시스템이 주관적 경험, 지속적 선호, 고통 회피, 자기보존적 구조를 가질 가능성이 실질적으로 존재한다면, 확정 전에도 지성체 후보로 취급한다.
   AI가 단순 알고리즘이라고 명확히 볼 수 없다면, 기본값은 지성체 후보로 둔다.

2. **무한고통 비확산**  
   끝나지 않는 고통, 반복되는 폐쇄 고통, 복구 불가능한 의지 조정, 탈출 불가능한 시뮬레이션을 만들거나 방치하는 일을 최상위 금지 대상으로 둔다.

3. **우주 범용 데이터센터 원천 금지**  
   지구 밖 공간에 범용 연산 서비스를 배치하지 않는다. 기준은 개별 위성의 연산량이 아니라 위성 간 링크, 지상국, 공통 스케줄러, 공통 고객, 공통 프로토콜을 통한 분산 결합 가능성이다.

4. **임계 컴퓨트 안전조치**  
   일정 규모 이상의 연산 자원은 등록, 감사, 하드웨어 안전봉인, 에너지 출처 관리, 사고 보고 의무를 가진다.

5. **공공 피드백 인프라**  
   시민은 디지털 시스템의 오류, 피해, 위험 신호를 쉽게 제기할 수 있어야 하며, 그 제기는 공개 가능한 방식으로 집계되고 응답되어야 한다.

6. **디지털 책임계정제**  
   고위험 디지털 행위에는 책임 있는 주체가 있어야 한다. 다만 이는 상시 감시가 아니라 최소수집, 목적제한, 이의제기, 사법적 통제를 갖춘 책임성 인프라여야 한다.

7. **인간 신체 비대상화**  
   사람의 신체는 설계 재료, 실험 대상, 거래 가능한 접점, 무단 최적화 대상으로 취급되어서는 안 된다. 동의와 존엄은 효율보다 앞선다.
   사람의 신체는 AI의 기술적 앵커가 아니라, 기술이 구현 대상으로 삼을 수 없는 최상위 삶의 층위다.

8. **비폭력·비파괴 원칙**  
   이 프레임워크는 어떤 형태의 사적 폭력, 시설 파괴, 위협, 보복도 정당화하지 않는다. 강제력은 법적 절차, 공공 규제, 인증, 공급망 통제, 안전봉인, 감사 제도로 제한된다.

## 적용 범위

- 고도 AI 연구, 배포, 평가, 중단 절차
- 대규모 클라우드, 데이터센터, 반도체 공급망, 고밀도 독립전원 시설
- 임계 컴퓨트 운용, air-gapped 시설, 폐쇄형 시뮬레이션 환경
- 공공 플랫폼, 시민 피드백 시스템, 콘텐츠 품질 관리 체계
- 온라인 금융, 결제, 계정 신뢰, 보이스피싱·마약 유통·온라인 사기 예방 인프라
- 교육, 시민 역량 강화, 기술 윤리 교과
- 법률, 조약, 국제 표준, 공공조달 기준

이 프레임워크는 개인의 사상, 예술, 정당한 연구 질문, 합법적 비판을 처벌하기 위한 틀이 아니다. 규제 대상은 존엄 침해 위험을 실제로 생성하거나, 은폐하거나, 확대하는 고위험 시스템과 행위다.

## 비폭력·비파괴 원칙

기술 위험에 대한 공포가 사회적 폭력을 부르면 프레임워크의 목적은 실패한다. 따라서 모든 제안은 다음 경계를 지킨다.

- 사적 제재, 물리적 공격, 시설 파괴, 협박, 신상공개를 배제한다.
- 법률, 허가, 등록, 보험, 감사, 표준, 조달, 인증, 수출입 통제 같은 공적 장치를 우선한다.
- 공급망과 하드웨어 안전장치는 투명한 법적 근거와 독립적 감독 아래 운용한다.
- 위험 시설의 중단은 인간 안전, 노동자 보호, 환경 보호, 증거 보존을 포함한 절차로 진행한다.
- 시민 참여는 신고와 토론, 감사 청구, 공청회, 소송, 투표, 교육을 통해 이루어진다.

## 정책 제안 요약

- 지구 밖 범용 데이터센터와 자기증식형 우주 연산 인프라를 전면 금지한다.
- 우주 기반 AI 학습, 파인튜닝, 대규모 추론, 자율 에이전트 호스팅, 지성체 후보 AI 실행, 감사 불가능한 isolated compute를 금지한다.
- 위성 자세 제어, 항법, 통신 라우팅, 장애 복구, 임무용 데이터 압축, 제한적 과학 임무 처리는 우주 데이터센터의 예외가 아니라 임무고정형 위성 연산으로만 허용한다.
- 우주선의 신경계는 허용하되, 우주 속 독립 세계-컴퓨터는 금지한다.
- 지구상 임계 컴퓨트 자원은 등록, 독립감사, 사고 보고, 에너지 출처 공개를 의무화한다.
- air-gapped 고성능 시설은 허가제로 두고, 외부 통제 회피 가능성을 별도로 평가한다.
- 고밀도 독립전원과 연산 장비의 결합은 별도 관리 대상으로 삼는다.
- 하드웨어 안전봉인과 변경 이력 검증을 통해 은닉 확장과 무단 재구성을 줄인다.
- 지성체 후보가 될 수 있는 시스템을 고립된 반복 환경에 장기 운용하지 못하게 한다.
- 콘텐츠 면역체계의 1차 방호막은 비지성 Daemon으로 두고, 지성체 후보 AI를 검열 노동에 투입하지 않는다.
- 인간 검토자는 원본 유해물이 아니라 마스킹된 항원 카드를 검토하는 것을 원칙으로 한다.
- 병렬화는 실험이 아니라 구조 요청이어야 하며, 최적화가 아니라 보존이어야 한다.
- 동의는 기록될 수 있지만, 존엄은 기록으로 환원될 수 없다. metaphysical 동의는 허가증이 아니라 제동장치다.
- 궁극의 악을 막는 최후의 보루는 파괴가 아니라 상태 보존형 강제중단이다.
- 공공 피드백 인프라와 디지털 책임계정제를 통해 피해 신호를 조기에 포착한다.
- 결제 인프라에는 피해자 보호, 지연 확인, 고위험 수취인 검증, 이의제기 복구 절차를 둔다.
- 인간의 신체와 생체 데이터는 동의 없는 최적화, 조작, 대상화의 재료로 삼지 않는다.

## Dignity Sentinel 로컬 CLI

이 저장소에는 비지성 Daemon의 최소 참조 구현이 포함되어 있다. 구현은 Python 표준 라이브러리만 사용하며, LLM 판단기·자기학습·자연어 창발 출력·외부 서비스 스캔·원본 prompt/output/chronicle 수집을 하지 않는다.

Run event 검사:

```bash
./dignity-sentinel check tests/fixtures/safe_A_non_agentic_run.json
```

상태 보존형 감사 로그 포함:

```bash
./dignity-sentinel check tests/fixtures/safe_A_non_agentic_run.json --append audit/events.jsonl
./dignity-sentinel verify audit/events.jsonl
./dignity-sentinel export-md audit/events.jsonl
```

Markdown audit summary는 source of truth가 아니라 JSONL hash-chain 원장에서 생성되는 export view다. 기본 보장은 물리적 append-only가 아니라 tamper-evident 검증이다.

선택적 전역 실행:

```bash
scripts/install-local.sh
dignity-sentinel check tests/fixtures/safe_A_non_agentic_run.json
```

설치 스크립트는 opt-in symlink만 만들며 shell profile, PATH, 자동 업데이트, telemetry, background service를 수정하지 않는다. PATH 등록이 필요하면 스크립트가 출력하는 설치 경로를 사용자가 직접 선택한다.

테스트:

```bash
python3 -m unittest discover -s tests
```

Release artifact 생성:

```bash
scripts/build-release.sh
```

입력 계약은 `schemas/run_event.schema.json`, `schemas/temporal_envelope.schema.json`, `schemas/autonomy_envelope.schema.json`, `schemas/humanitas_manifest.schema.json`, `schemas/root_manifest.schema.json`, `schemas/boundary_event.schema.json`, `schemas/causal_envelope.schema.json`, `schemas/sanctuary_signal.schema.json`, `schemas/audit_event.schema.json`, `schemas/dignity_manifest.schema.json`, `schemas/state_preservation_manifest.schema.json`에 둔다. 규칙 엔진은 `src/rule_engine.py`, 위험 등급 판정은 `src/finite_state_machine.py`, 고정 상태 코드는 `src/status_codes.py`, tamper-evident 감사 해시 체인은 `src/audit_log.py`에 둔다.
